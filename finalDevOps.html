DevOps Complete Beginner Roadmap

(with Topics, Commands, and Practice Tasks)

1. Linux Fundamentals

Learning:

Understand Linux file system structure and navigate it using commands like ls, cd, pwd
Perform file operations: create, delete, move, copy with mkdir, rm, mv, cp
Manage users and groups using useradd, usermod, groupadd, passwd
Understand file permissions and change them using chmod, chown
Manage software using package managers: apt, yum, dnf
Network commands: ping, netstat, curl, ifconfig, ip, traceroute
Monitor and control processes: ps, top, htop, kill, killall
Use redirection >, >>, < and piping |
Start/stop services using systemctl start|stop|status|enable <service>
Set scheduled jobs using crontab -e, list with crontab -l
Search and manipulate text with grep, awk, sed, find, locate

Tasks:

1 Create a new user, set their password, and add them to a group.
2 Compress a given directory into a .tar.gz archive using the tar command.
3 Set a cron job to clean .log files in a folder every hour (use a simple rm command, no script).
4 Enable and start the SSH service, then verify its status.

2. Networking Basics for DevOps

Learning:

OSI Model basics and ports
IP addressing, DNS, DHCP
NAT, private vs public IP
Common protocols: HTTP, HTTPS, FTP, SSH, SCP
Tools: ping, traceroute, netstat, ss, curl, wget, telnet, nc
Host file and DNS resolution
Check open ports using nmap, ss, netstat

Tasks:

1 Use ping, traceroute, and nslookup to troubleshoot a domain name.
2 Check if a port is open on a server using nc or telnet.
3 Set up a simple HTTP server with Python and test connectivity.
4 Scan local network ports using nmap.

3. Version Control with Git

Learning:

Initialize repositories with git init, or clone existing ones with git clone
Track changes with git status, stage with git add, commit with git commit
Sync changes using git push, git pull, git fetch
Work with branches: create git branch, switch git checkout, merge git merge
Resolve merge conflicts and view history with git log, git diff
Use git tag for versioning, configure .gitignore, and set global user config
Collaborate using GitHub/GitLab: push code, open pull requests, fork repos

Tasks:

1 Create and push a Git repo to GitHub with README and .gitignore
2 Create a feature branch, make changes, and merge to main after conflict resolution
3 Write a .gitignore for a sample Node.js project
4 Clone a public GitHub repo, make a change, and submit a pull request

4. Shell Scripting

Learning:

Write bash scripts with #!/bin/bash and run them with chmod +x
Use variables, read user input, print with echo
Conditional statements: if, else, elif
Loops: for, while, until
Functions and return values
Access script arguments via $1, $2, etc.
Check exit status with $?

Tasks:

1 Create a script that prints the number of lines, words, and characters in a file
2 Write a script that monitors and restarts a service if it's down
3 Automate compression of .log files in a folder using a loop
4 Create a script that accepts input and outputs a personalized greeting

5. Docker and Containerization

Learning:

What is Docker? Containers vs VMs
Install Docker, understand Docker architecture
Basic commands: docker run, ps, stop, rm, images, rmi
Create and use Dockerfiles
Volumes and bind mounts
Docker Compose basics and file structure
Networking in Docker: bridge, host, none
Pushing to and pulling from Docker Hub

Tasks:

1 Run a basic Nginx container and expose it on a port
2 Write a Dockerfile for a Python Flask app and build an image
3 Create a docker-compose.yml file for a multi-container app (e.g., web + db)
4 Push a custom image to Docker Hub

6. CI/CD with GitHub Actions

Learning:

What is CI/CD?
GitHub Actions basics: workflows, jobs, steps
Writing .github/workflows/main.yml
Triggers: push, pull_request, schedule
Using marketplace actions
Setting environment variables and secrets
Caching dependencies

Tasks:

1 Create a workflow that runs tests on every push
2 Setup automatic deployment on push to main branch
3 Add environment variables and use them in your workflow
4 Use GitHub Actions to lint a codebase (e.g., ESLint, flake8)

7. Infrastructure as Code - Terraform Basics

Learning:

What is IaC?
Install Terraform
Create and initialize configuration files (.tf)
Providers and resources
Plan, apply, destroy cycle
Output variables and remote state basics

Tasks:

1 Deploy an EC2 instance using Terraform on AWS
2 Create a basic VPC and subnet configuration
3 Use variables and outputs to make your config modular
4 Destroy and reapply infrastructure safely

8. Configuration Management with Ansible

Learning:

What is Ansible?
Install Ansible
Inventory files and ad-hoc commands
YAML & Playbooks
Modules: apt, copy, template, service, user
Handlers and conditionals

Tasks:

1 Write a playbook to install and start Nginx on a remote host
2 Create a user and set SSH key via playbook
3 Use a template to configure a web server
4 Run an ad-hoc command to check uptime on multiple machines

9. Monitoring and Logging

Learning:

Importance of monitoring in DevOps
Install and use Prometheus for metrics
Setup Grafana dashboards
Node Exporter and basic metrics
Logging with ELK stack (Elasticsearch, Logstash, Kibana)
Alerting basics

Tasks:

1 Install Prometheus and configure it to monitor Node Exporter
2 Create a Grafana dashboard with at least 2 visualizations
3 Set up an alert rule for high CPU usage
4 Install and visualize logs using the ELK stack

10. Kubernetes Basics

Learning:

What is Kubernetes and why use it?
Install Minikube or use Play with Kubernetes
Pods, Deployments, Services
YAML files for resource definitions
Basic kubectl commands: get, describe, apply, delete
Port forwarding and service exposure

Tasks:

1 Deploy a pod with Nginx and expose it
2 Create a deployment and scale it up/down
3 Write a YAML manifest for a service + deployment
4 Use kubectl port-forward to test a deployed pod

11. Cloud Fundamentals (AWS)

Learning:

What is Cloud? IaaS vs PaaS vs SaaS
AWS core services: EC2, S3, IAM, VPC, RDS
IAM roles and permissions
AWS CLI setup and usage
Budgeting and cost estimation tools

Tasks:

1 Launch an EC2 instance manually and connect via SSH
2 Upload files to an S3 bucket using AWS CLI
3 Create and attach an IAM role to an instance
4 Use the AWS pricing calculator for estimating service costs
